What is Server Side Rendering (SSR)?

Have you ever encountered with the word SSR? It stands for Server Side Rendering in web development terminology. But what does Server Side Rendering actually mean? In order to understand the meaning and working principle of SSR, a person should have at least a basic knowledge of website development and web page rendering. Lets start with a short explanation about web rendering.



What is a web rendering?

When we enter the domain of any website on the address bar of our web browser and try to reach the website, the browser send an HTTP request to the web server where the website is hosted. It is usually a GET request including a site URL and a couple of headers.

After this request reaches the web server, the server sends a response based on our GET request. When this response is rendered on our browser screen, then we can see the web page we wish to go on our monitor. This process is simply called a web rendering.



Two types of web rendering.

There are two types of web rendering: Server Side Rendering (SSR) and Client Side Rendering (CSR). But what is a server and client? A server is simply the web server where the website is hosted. A client is a web browser which is used to connect to the website. In Client Side Rendering, the web rendering process is happening on the client side by a web browser.

The browser gets a response from the web server and renders it on the screen for us. In Server Side Rendering, web rendering happens in the server side and our web browser gets a rendered web page to show it on the monitor.



Common properties of Client Side Rendering.

- Rendered in the web browser side.

- Uses a lot of JavaScript queries and functions on the front end.

- Usually, a web framework or library is used for front end.

- Used when front side information processing, data visualization, events, or actions required.

- A client side web framework is used to control the communication between the client and the server.





Common properties of Server Side Rendering.

- Rendered in the web server where the website is hosted.

- Mostly uses static templates for web page rendering.

- Minimal front end programming is required.

- All information rendered on the browser screen is sent by the web server such as data visualization or static files like images.

- The server is responsible for controlling all the HTTP requests and sending responses for those requests.



The working principle of SSR.

The working principle of SSR is very simple. An online user sends an HTTP request with his / her web browser and this request is received by the web server. Any input value or form submission data got from this request is processed on the web server level with the help of the programming language was used to develop this web application.

After that, an HTTP response is sent back with the relevant output value and data received from the information processing. This response is rendered on the web server before it is sent to the corresponding web browser.



What are some popular examples of web frameworks that implement Server Side Rendering?

WordPress CMS: The most popular content management system in the world is using server side rendering method for the delivery of its exceptional and unique content.

Python Django Framework: Django framework is a Python based full stack web framework which helps you to easily built your web application. It adopts server side rendering as a web rendering method to offer fast website loading times and higher SEO ranking for your website. Its technology receives an HTTP request, directs it to the matching URL and picks the relevant HTML template to send as a response to the HTTP request.

Vue.js JavaScript Framework: Vue.js is a JavaScript front end web framework to be used to apply server side rendering method to web applications built on different backend technologies. You can easily use it with Laravel PHP framework, or with NodeJS to make your web application to be rendered on the web server side.



Why is SSR so important for your SEO ranking?

Server Side Rendering is very important for SEO ranking due to the fact that even the most modern search engines like Google, Bing, Yahoo, and Yandex have difficulties in crawling the websites prefer client side rendering as their web rendering method.

It is because the crawl bots do not use a web browser to render a web page before they can crawl it. Therefore, crawlers usually read a client side rendered website as a blank web page. And this situation kills your search engine results page (SERP) ranking for the obvious reason.

If a web crawler can not read your website, then it can not send the meta data and the content of your website to the search console. As a result, your web pages are not indexed and thus, they are not shown on the search engine results pages.
