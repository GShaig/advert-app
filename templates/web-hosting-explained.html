What is Web Hosting?




What is a web hosting and why does a website need it?

A web hosting is where your website lives in. In other words, where all your website data included static files, applications, servers and configuration files stored.

There are mainly four types of web hosting:

Shared web hosting: In this option, you share the same server with other website owner. The number of tenants depends on the hosting company and available resources. It is usually the cheapest option and suitable for individuals or small business owners.

VPS (Virtual Private Server) web hosting: This hosting type gives you a little more freedom and customization. You can install your packages to this type of server or upload your configuration files to the server. This option usually offered to developers and individuals who need a higher performance than shared hosting.

Dedicated server web hosting: This option is like renting the entire apartment yourself. You can do whatever you want to do with your server and most probably you will hire a team of developers to manage that server. Usually big companies and corporations prefer a dedicated server hosting in order to achieve high performance and security.

Cloud web hosting: Cloud hosting providers usually allocate resources for you according to your web hosting needs. You can use different services with as per your project requirements and usually these services are provided in container environments. Cloud hosting is usually associated with a pay as you go payment structure.

Think about sharing your flat, renting a separate unit or renting the entire building.

If your website project is very big and complex, you should definitely go with VPS or dedicated server hosting. VPS server is like renting a portion of a dedicated server. That is why dedicated and VPS hosting options are more secure compared with a shared hosting.

Hosting provider plays an important role for your website security and performance. Customer support is also important, as you'll want to ensure that you have access to help and assistance if you encounter any issues with your website. And scalability is important to consider if your website is expected to grow in the future.

Connecting your domain with your web hosting?

If you bought your domain name and your web hosting plan from the same service provider, directing your domain to your website is pretty straight forward.

You just need to add a couple of DNS records to the DNS settings of your domain name and you are good to go! Even some service providers do this process automatically, in a click of a button.

But if your domain name provider and web hosting plan provider are two different companies, you need to do a bit of work yourself. It is still not that hard, but you have to adjust some settings in order to get your domain name connected.

You first need to go to your domain name provider and change your domain settings by adding your web hosting provider's name server records to your domain settings. Then go back to your web hosting provider and add all the required DNS records there. After that, you will be able to redirect your domain name to your website.





What are the most common web server OS (Operating Systems)?

Linux is the most widespread operating system for web servers. Approximately 80% of all web servers use Linux and different Linux distributions.

Some popular Linux distributions for web servers are Ubuntu, CentOS, and Debian. Due to the fact Linux running every 4 out of 5 web servers, web developers are required to have extensive Bash command line knowledge in order to manage a server properly.

Windows shares roughly 20% of the market. As you can easily guess, people do not choose a Windows server if they do not have a specific reason to do so.

Companies usually choose a Windows server if they have entire ecosystem run on Windows and their developers are more familiar with Windows operating system such as using PowerShell command line.



What is root access for web hosting?

A root access is an administrator privilege that comes with your hosting plan. But not all hosting plans and service providers include it in the package.

Root access means that you can connect to your web hosting server remotely as a root (admin) user. Usually it is done by SSH (Secure Shell Protocol) and called Shell Access in daily life.

On SSH protocol, you connect to your web server by entering your username and password provided by your web hosting service provider. If that username and password belongs to the root user of your web server, we call that the service provider has given you a root access.

Then you manage your web server with the help of Bash commands (if it is a Linux server) or PowerShell (if it is a Windows). Normally, a root access available for all web hosting plans mentioned above, except the shared hosting.

If you have a shared hosting plan, that means you share your web server with the other clients so giving root access to you can obviously cause security and privacy concerns for the other users sharing the server with you. That is why most of the shared hosting providers create additional user for each tenant and do not provide root access in order to prevent the users from interfering with the web server.
